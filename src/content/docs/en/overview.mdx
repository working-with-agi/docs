## The Problem

You want to add AI-powered developer tools to your product — a terminal with AI assistance, or intelligent code search that understands intent. Building these from scratch takes months and requires expertise in PTY handling, vector databases, embedding models, and real-time WebSocket protocols.

## The Solution

WorkWithAGI SDK gives you two production-ready capabilities as drop-in components:

### AI Terminal

A web-based terminal that runs in any browser. Your users get:

- Full PTY terminal (not a fake shell)
- Built-in AI assistant that understands context
- Command threat detection and filtering
- Session recording and playback
- Customizable themes and keybindings

Embed it with an iframe or the JavaScript SDK:

```html
<!-- iframe: zero config -->
<iframe src="https://terminal.workwithagi.dev/embed?key=YOUR_KEY"></iframe>
```

```typescript
// SDK: full control
import { WorkAGI } from "@working-with-agi/sdk";

const term = WorkAGI.terminal("#container", {
  theme: "dark",
  ai: true,
  recording: true,
  onThreat: (alert) => notify(alert),
});
```

### RAG Code Search

Index any codebase and let users search by describing what code does — not by remembering exact names:

```typescript
const search = WorkAGI.codeSearch({
  index: "my-project",
  embedModel: "bge-small-en-v1.5",
});

// Index a directory
await search.index("./src");

// Search by intent
const results = await search.query("error handling for API timeouts");
// → file, line number, relevance score, code snippet
```

Features:
- Automatic incremental indexing (file watcher)
- OpenVINO-accelerated embeddings
- Qdrant vector storage (self-hosted or managed)
- Call graph tracing (find all callers/callees of a function)

## Architecture

```
Your App
  │
  ├── WorkAGI.terminal()  →  WebSocket (Socket.IO) → PTY + AI Agent
  │
  └── WorkAGI.search()    →  REST API → Embedding → Qdrant Vector DB
```

Both components are self-hostable. No data leaves your infrastructure unless you choose our managed service.
