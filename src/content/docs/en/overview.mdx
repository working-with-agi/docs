## The Problem

You want to add AI-powered developer tools to your product — a terminal with AI assistance, or an intelligent knowledge search that understands intent. Building these from scratch takes months and requires expertise in PTY handling, vector databases, embedding models, and real-time WebSocket protocols.

## The Solution

WorkWithAGI SDK gives you two production-ready capabilities as drop-in components:

### AI Terminal

A web-based terminal that runs in any browser. Your users get:

- Full PTY terminal (not a fake shell)
- Built-in AI assistant that understands context
- Command threat detection and filtering
- Session recording and playback
- Customizable themes and keybindings

Embed it with an iframe or the JavaScript SDK:

```html
<!-- iframe: zero config -->
<iframe src="https://terminal.workwithagi.dev/embed?key=YOUR_KEY"></iframe>
```

```typescript
// SDK: full control
import { WorkAGI } from "@working-with-agi/sdk";

const term = WorkAGI.terminal("#container", {
  theme: "dark",
  ai: true,
  recording: true,
  onThreat: (alert) => notify(alert),
});
```

### KnowledgeHub

Index documents, code, and wikis — then let users search by describing what they need, not by remembering exact names:

```typescript
const hub = WorkAGI.knowledgeHub({
  index: "my-project",
});

// Index a directory
await hub.ingest("./src");

// Search by intent
const results = await hub.search("error handling for API timeouts");
// → source, section, relevance score, snippet
```

Features:
- Multi-source ingestion (code, docs, wikis)
- Automatic incremental indexing (file watcher)
- Semantic search across your entire knowledge base
- Context-aware retrieval with source attribution

## Architecture

```
Your App
  │
  ├── WorkAGI.terminal()       →  WebSocket (Socket.IO) → PTY + AI Agent
  │
  └── WorkAGI.knowledgeHub()   →  REST API → Embedding → Vector DB
```

Both components are self-hostable. No data leaves your infrastructure unless you choose our managed service.
