The Knowledge Base provides retrieval-augmented generation (RAG) for your AI terminals and agents. Upload documents, connect external sources, and let the AI search your knowledge in real time.

## KnowledgeClient

The `KnowledgeClient` manages document ingestion, semantic search, and context retrieval.

```typescript
import { KnowledgeClient } from "@work-with-ai/sdk";

const knowledge = new KnowledgeClient(
  "https://api.workwithagi.dev",
  "your-api-key",
);
```

### Upload Documents

Upload a `File` or `Blob` to the knowledge base. Optionally provide a document ID to replace an existing document.

```typescript
// Upload a File (e.g. from an <input> element)
const result = await knowledge.upload(file);
console.log(result.document_id); // "doc_abc123"
console.log(result.filename);    // "architecture.md"
console.log(result.chunk_count); // 42

// Upload with an explicit document ID (replaces if exists)
await knowledge.upload(file, "my-custom-id");

// Upload a Blob
const blob = new Blob(
  ["The deployment process requires approval from two reviewers."],
  { type: "text/plain" },
);
await knowledge.upload(blob, "team-guidelines");
```

### Search

Semantic search returns relevant chunks ranked by relevance score:

```typescript
const response = await knowledge.search("how to handle authentication errors");

console.log(response.query); // "how to handle authentication errors"
for (const hit of response.results) {
  console.log(`[${hit.score.toFixed(2)}] ${hit.filename} (chunk ${hit.chunk_index})`);
  console.log(hit.chunk);
}
// → [0.92] auth.md (chunk 3)
//   When an authentication error occurs, the middleware returns a 401...
```

You can limit the number of results and filter by document ID:

```typescript
// Return at most 5 results, scoped to a specific document
const response = await knowledge.search("deployment steps", 5, "runbook-doc-id");
```

### Context for AI

Retrieve pre-formatted context suitable for passing directly to an AI model:

```typescript
const response = await knowledge.context("explain our caching strategy", 5);

console.log(response.query);   // "explain our caching strategy"
console.log(response.context); // Combined text from the top matching chunks
```

### Remove Documents

Delete a document and all its indexed chunks:

```typescript
await knowledge.remove("doc_abc123");
```

## External Sync

Connect external document sources for automatic synchronization. Documents are re-indexed on a configurable schedule.

```typescript
// Google Drive
await knowledge.connect({
  provider: "google-drive",
  name: "Engineering Docs",
  config: { folder_id: "1a2b3c4d5e", service_account_key: "..." },
  schedule: "0 */6 * * *", // Every 6 hours
});

// SharePoint
await knowledge.connect({
  provider: "sharepoint",
  name: "Engineering Site",
  config: {
    tenant_id: "...",
    client_id: "...",
    client_secret: "...",
    site_url: "https://company.sharepoint.com/sites/engineering",
    library_name: "Documents",
  },
  schedule: "0 2 * * *", // Daily at 2 AM
});

// Notion
await knowledge.connect({
  provider: "notion",
  name: "Product Wiki",
  config: { integration_token: "...", database_ids: "abc123,def456" },
  schedule: "0 */4 * * *", // Every 4 hours
});

// Confluence
await knowledge.connect({
  provider: "confluence",
  name: "Eng & Ops Spaces",
  config: {
    api_token: "...",
    email: "...",
    base_url: "https://company.atlassian.net",
    space_keys: "ENG,OPS",
  },
  schedule: "0 3 * * *",
});
```

<Aside type="tip">
  All credentials are encrypted at rest. For self-hosted deployments, credentials are stored in your own Vault instance.
</Aside>

### Supported Sync Providers

`"google-drive"` | `"dropbox"` | `"notion"` | `"confluence"` | `"sharepoint"` | `"github"` | `"s3"`

### Managing Sync Connections

```typescript
// List all connections
const connections = await knowledge.connections();
for (const conn of connections) {
  console.log(`${conn.name} (${conn.provider}): ${conn.status}, ${conn.doc_count} docs`);
  // → "Engineering Docs (google-drive): active, 128 docs"
}

// Trigger a manual sync
const result = await knowledge.sync("connection-id");
console.log(result.added, result.updated, result.removed);

// Pause / resume a connection
await knowledge.pause("connection-id");
await knowledge.resume("connection-id");

// Disconnect a source
await knowledge.disconnect("connection-id");
```

## Supported File Formats

| Category | Formats |
|---|---|
| **Code** | `.ts`, `.js`, `.py`, `.go`, `.rs`, `.java`, `.rb`, `.php`, `.c`, `.cpp`, `.cs`, `.swift`, `.kt` and 10+ more |
| **Documents** | `.md`, `.mdx`, `.txt`, `.rst`, `.adoc` |
| **Office** | `.pdf`, `.docx`, `.xlsx`, `.pptx` |
| **Data** | `.json`, `.yaml`, `.yml`, `.toml`, `.csv` |
| **Web** | `.html`, `.xml` |
| **Config** | `.env`, `.ini`, `.conf`, `Dockerfile`, `Makefile`, `docker-compose.yml` |

<Aside type="note">
  Binary files (images, videos, compiled binaries) are skipped automatically. Office files are converted to text before indexing.
</Aside>

## Standalone Usage Example

The `KnowledgeClient` can be used directly in any TypeScript or JavaScript application:

<Tabs>
  <TabItem label="Node.js">
  ```typescript
  import { KnowledgeClient } from "@work-with-ai/sdk";
  import { readFile } from "node:fs/promises";

  const knowledge = new KnowledgeClient(
    "https://api.workwithagi.dev",
    process.env.AGI_API_KEY,
  );

  // Upload a file as a Blob
  const buffer = await readFile("./docs/architecture.md");
  const blob = new Blob([buffer], { type: "text/markdown" });
  const { document_id } = await knowledge.upload(blob, "architecture");

  // Search and use the context
  const { context } = await knowledge.context("how does authentication work?");
  console.log(context);
  ```
  </TabItem>
  <TabItem label="Browser">
  ```typescript
  import { KnowledgeClient } from "@work-with-ai/sdk";

  const knowledge = new KnowledgeClient(
    "https://api.workwithagi.dev",
    "your-api-key",
  );

  // Upload from a file input
  const input = document.querySelector<HTMLInputElement>("#file-input")!;
  input.addEventListener("change", async () => {
    const file = input.files?.[0];
    if (file) {
      const result = await knowledge.upload(file);
      console.log(`Uploaded ${result.filename}: ${result.chunk_count} chunks`);
    }
  });

  // Search
  const { results } = await knowledge.search("deployment process", 5);
  results.forEach((hit) => {
    console.log(`[${hit.score.toFixed(2)}] ${hit.filename}: ${hit.chunk}`);
  });
  ```
  </TabItem>
</Tabs>
