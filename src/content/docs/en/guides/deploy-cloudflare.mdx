Deploy your Next.js application to Cloudflare Workers using [OpenNext](https://opennext.js.org/cloudflare) for full SSR support, API routes, and edge performance.

## Overview

| Component | Role |
|---|---|
| **Next.js 16** | Application framework (App Router) |
| **@opennextjs/cloudflare** | Builds Next.js into Cloudflare Workers-compatible output |
| **wrangler** | Cloudflare CLI for deployment |
| **GitHub Actions** | CI/CD pipeline |

<Aside type="tip">
  OpenNext converts Next.js output (including API routes, middleware, and SSR) into a single Cloudflare Worker. No code changes are needed in your Next.js app.
</Aside>

## Prerequisites

- A Cloudflare account
- A GitHub repository for your Next.js project
- Node.js 22+

<Steps>
1. ### Install dependencies

   ```bash
   npm install @opennextjs/cloudflare
   ```

2. ### Create `open-next.config.ts`

   ```typescript
   import type { OpenNextConfig } from "@opennextjs/cloudflare";

   const config: OpenNextConfig = {
     default: {
       override: {
         wrapper: "cloudflare-node",
         converter: "edge",
         proxyExternalRequest: "fetch",
         incrementalCache: "dummy",
         tagCache: "dummy",
         queue: "dummy",
       },
     },
     edgeExternals: ["node:crypto"],
     middleware: {
       external: true,
       override: {
         wrapper: "cloudflare-edge",
         converter: "edge",
         proxyExternalRequest: "fetch",
         incrementalCache: "dummy",
         tagCache: "dummy",
         queue: "dummy",
       },
     },
   };

   export default config;
   ```

3. ### Create `wrangler.jsonc`

   ```json
   {
     "$schema": "node_modules/wrangler/config-schema.json",
     "name": "your-app-name",
     "main": ".open-next/worker.js",
     "compatibility_date": "2025-04-01",
     "compatibility_flags": ["nodejs_compat"],
     "assets": {
       "directory": ".open-next/assets",
       "binding": "ASSETS"
     }
   }
   ```

4. ### Update `next.config.ts`

   Remove `output: "standalone"` if present — Workers mode does not use the standalone output.

   ```typescript
   import type { NextConfig } from "next";

   const nextConfig: NextConfig = {
     // Do NOT set output: "standalone"
   };

   export default nextConfig;
   ```

5. ### Add npm scripts

   ```json
   {
     "scripts": {
       "build:worker": "npx @opennextjs/cloudflare build",
       "preview": "npx wrangler dev",
       "deploy": "npx wrangler deploy"
     }
   }
   ```

6. ### Local preview

   ```bash
   npm run build:worker
   npm run preview
   ```

   Open `http://localhost:8787` to verify.
</Steps>

## GitHub Actions CI/CD

### Set up Secrets

Add these to your repository **Settings → Secrets and variables → Actions**:

| Secret | Description |
|---|---|
| `CLOUDFLARE_API_TOKEN` | Create from Cloudflare Dashboard → **My Profile → API Tokens** using the "Edit Cloudflare Workers" template |
| `CLOUDFLARE_ACCOUNT_ID` | Found on your Cloudflare Dashboard overview page |

### Workflow

Create `.github/workflows/deploy.yml`:

```yaml
name: Deploy to Cloudflare Workers

on:
  push:
    branches: [main]
  workflow_dispatch:

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 22

      - run: npm ci

      - run: npx @opennextjs/cloudflare build

      - uses: cloudflare/wrangler-action@v3
        with:
          apiToken: ${{ secrets.CLOUDFLARE_API_TOKEN }}
          accountId: ${{ secrets.CLOUDFLARE_ACCOUNT_ID }}
          command: deploy
```

Push to `main` and the workflow will automatically build and deploy.

## Custom Domain

After your first deploy, configure a custom domain:

1. Go to **Cloudflare Dashboard → Workers & Pages → your worker → Settings → Domains & Routes**
2. Click **Add** → **Custom Domain**
3. Enter your domain (e.g., `app.example.com`)
4. Cloudflare handles SSL automatically

Or add to `wrangler.jsonc`:

```json
{
  "routes": [
    { "pattern": "app.example.com", "zone_name": "example.com" }
  ]
}
```

## API Routes

Next.js API routes (`app/api/...`) work automatically on Cloudflare Workers via OpenNext. No additional configuration is needed.

```
app/
  api/
    v1/
      users/
        route.ts    → https://your-app.workers.dev/api/v1/users
      health/
        route.ts    → https://your-app.workers.dev/api/v1/health
```

## Environment Variables

Use `wrangler.jsonc` for non-secret values and Cloudflare Dashboard for secrets:

```json
{
  "vars": {
    "NEXT_PUBLIC_APP_URL": "https://app.example.com"
  }
}
```

For secrets:
```bash
npx wrangler secret put AUTH_SECRET
```

## Troubleshooting

| Issue | Solution |
|---|---|
| `Pages project` error with `wrangler deploy` | Ensure `wrangler.jsonc` uses `main` (Workers), not `pages_build_output_dir` (Pages) |
| `output: "standalone"` build failure | Remove `output: "standalone"` from `next.config.ts` |
| `node:crypto` error | Add `edgeExternals: ["node:crypto"]` to `open-next.config.ts` |
| Large bundle warnings | Enable `nodejs_compat` in `compatibility_flags` |
