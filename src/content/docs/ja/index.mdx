## あなたのサービスに、AIの力を

ユーザーがAIに自然言語で指示を出し、AIがデータを読み取り、分析し、結果を返す——そんな体験を、あなたのプロダクトに数行のコードで追加できます。

## プロダクト開発の初期から組み込む理由

WorkWithAGI をプロジェクトの初期段階から導入することで、**ユーザーに価値を届けながら、次に何を作るべきかが見えてきます。**

<CardGrid>
  <Card title="ユーザーと対話しながら開発を進める" icon="rocket">
    システムが未完成でも、AIがユーザーの質問に答え、足りない機能を補完します。MVP段階からAIが搭載されていることで、ユーザーは「とりあえず使ってみる」ハードルが下がります。
  </Card>
  <Card title="ユーザーの操作から次の開発アイテムを抽出" icon="magnifier">
    AIとのやりとりは記録されます。ユーザーが何を聞き、何に困り、どんな操作を試みたか——その行動データから、**次に開発すべき機能やUI改善のヒント**を自動的に抽出できます。
  </Card>
</CardGrid>

```
プロダクト開発サイクル

  ┌─ ユーザーがAIに指示 ──────────────────┐
  │  「〇〇のレポートを出して」              │
  │  「△△の設定を変えたい」                │
  │  「□□ってできないの？」                │
  │                                        │
  ▼                                        │
  AIが対応（できる範囲で）                   │
  │                                        │
  ▼                                        │
  対話ログの分析 ──────────────────────────│
  │  ・頻出リクエスト → 次の機能候補        │
  │  ・AI対応できなかった操作 → UI改善点    │
  │  ・ユーザーの言葉遣い → UXライティング  │
  │                                        │
  ▼                                        │
  開発チームにフィードバック ───→ 次のスプリント
```

### 具体例: ダッシュボードに必要な情報が足りないケース

あなたのサービスにダッシュボードがあるとします。リリース直後は最低限のグラフとテーブルだけ。ユーザーは必要な情報を見つけられず、離脱してしまうかもしれません。

**AIが組み込まれていると：**

```
ユーザー: 「先月と今月の売上を比較したい」
AI: 「ダッシュボードに月次比較機能はまだありません。
      代わりにデータを集計しました。先月: ¥1,240万、今月: ¥1,380万（+11.3%）」

ユーザー: 「チームごとの内訳も見たい」
AI: 「チーム別の表示機能は現在開発中です。
      手元のデータから集計すると…（結果を表示）」
```

**開発チームが得られるインサイト：**
- 「月次比較」が頻出リクエスト → **次スプリントでダッシュボードにグラフ追加**
- 「チーム別内訳」の需要が高い → **フィルター機能の優先度を上げる**
- ユーザーは「売上」ではなく「比較」を求めている → **UI設計のヒント**

AIがなければ、ユーザーは黙って離脱し、何が不足していたか分かりませんでした。AIとの対話ログが、**ユーザーの声なき要望**を可視化します。

## どんなサービスに使えるか？

- **SaaS プロダクト** — ユーザーがAIと対話しながら作業できるUIを追加
- **社内ツール** — 社員が自然言語でデータに問いかけるインターフェース
- **カスタマーサポート** — AIがナレッジベースから回答を自律生成
- **データ分析** — 「売上の傾向を教えて」で即座にレポート生成
- **教育プラットフォーム** — 学習者がAIアシスタントと対話しながら課題に取り組む
- **新規プロダクト** — MVP段階からAIを搭載し、ユーザー行動から次の開発を導く
