## システム概要

Work with AI は **Aether Platform** 上に構築されています。Kubernetes 上で動作する **AGI Terminal** が、様々なサービスからの AI ワークロードをスケーラブルに処理します。

## レイヤー構成

### フロントエンドレイヤー

あなたのアプリケーションに `@work-with-ai/vue` 等のフレームワークコンポーネントを配置します。ユーザーはソフトウェアのようなインターフェースから AI との作業を開始します。

- Vue / React / Angular 対応
- WebSocket で AGI Terminal に接続
- xterm.js レンダリング（ユーザーには透過的）

### AGI Terminal (agiterm)

Aether Platform 上の AI エージェント実行環境です。セッションごとにサンドボックスコンテナが起動し、安全にエージェントを実行します。

**セッションのライフサイクル:**

1. **Provision** — サンドボックスコンテナ / ワークスペースを作成
2. **Inject** — スキル（SKILL.md）、コンテキスト（CLAUDE.md）、認証情報（`.agiterm/credentials/`, 0600）をワークスペースに注入
3. **Start** — AI CLI をワークスペース内で起動

```
{workspace}/
├── .claude/agents/skills/{provider}/
│   └── SKILL.md
├── .agiterm/credentials/
│   ├── token     (0600)
│   └── api_url   (0600)
└── CLAUDE.md
```

### MAGI Engine

AGI Terminal 内で複数の AI エージェントが異なる視点から協調します。NATS メッセージングによるリアルタイムのエージェント間通信で合議を実現。

- **3つの視点**: 賛成 / 反対 / 中立
- **合議**: 多角的に検証された結論を導出
- **NATS 基盤**: エージェント間のリアルタイム通信

### Knowledge Hub

ドキュメントやデータを知識基盤として管理するサブシステムです。

| コンポーネント | 役割 |
|------------|------|
| **Ingestion** | ドキュメント・コード・Wiki の取り込み |
| **Embedding Engine** | テキストをベクトルに変換 |
| **Qdrant** | ベクトルデータベース |
| **Semantic Search** | 意味ベースの検索 |

Knowledge Hub が「AI が何を知っているか」を提供します。

### Skill Registry

Knowledge Hub の上で動くスキルを登録・管理します。スキルは `SKILL.md` ファイルとして定義され、セッション作成時にワークスペースに注入されます。

Skill Registry が「AI が何をできるか」を定義します。AI は SKILL.md の手順に従ってシステムを操作します。

### Kubernetes Infrastructure

AGI Terminal は Kubernetes 上でスケールします。

- **マルチテナント分離** — テナントごとにリソース制限とネットワーク分離
- **オートスケーリング** — 需要に応じて AGI Terminal Pod を自動スケール
- **NATS** — 内部通信のみ（インターネット非公開）

## セキュリティ

### コンテナサンドボックス隔離

各 AGI Terminal セッションは専用のコンテナサンドボックスで実行されます。これにより、セッション間およびテナント間の強力な隔離が実現されます。

**隔離の境界:**
- **ファイルシステム隔離** — 各セッションは独自のファイルシステムを持ちます。あるセッションで作成されたファイルは他のセッションからは見えません
- **ネットワーク分離** — ネットワークポリシーによりセッション間通信を制限。セッションはインターネットと承認済み内部サービスにのみアクセス可能
- **リソース制限** — Kubernetes リソースリミットにより、セッションごとに CPU、メモリ、ストレージのクォータを適用
- **プロセス隔離** — 各コンテナは最小限の権限で非 root ユーザーとして実行（`CAP_SYS_ADMIN` なし、`CAP_NET_RAW` なし）

**セッションライフサイクルのセキュリティ:**
1. 攻撃面を最小化した堅牢化済みベースイメージからコンテナを作成
2. 認証情報は `0600` パーミッションのファイルとして注入（セッションユーザーのみ読み取り可能）
3. セッション終了時にセキュアなクリーンアップを実行 — 認証情報は消去、コンテナは破棄

**ランタイムセキュリティ:**
- **読み取り専用ルートファイルシステム** — 指定された書き込み可能パス（`/workspace`, `/tmp`）のみ書き込み可能
- **権限昇格の禁止** — セキュリティコンテキストで `allowPrivilegeEscalation: false`
- **Seccomp プロファイル** — デフォルトまたはカスタムの seccomp プロファイルでシステムコールを制限
- **ネットワークポリシー** — Kubernetes NetworkPolicy で Egress を制御（セッション間のラテラルムーブメント防止）

### 認証情報管理

外部サービスの認証情報（API キー、トークン）はコンテナイメージに埋め込まれません:
1. Vault（またはシンプルなデプロイでは Kubernetes Secrets）に保存
2. セッション開始時に `.agiterm/credentials/` に `0600` パーミッションで注入
3. セッションごとにスコープ — 各セッションは必要な認証情報のみ取得
4. セッション終了時に自動的に失効/クリーンアップ

### 監査とコンプライアンス

- すべてのターミナル I/O はコンプライアンスとフォレンジック分析のために記録可能
- セッション記録は暗号化保存され、改ざん検知が可能
- API アクセスはテナント、ユーザー、セッション、タイムスタンプ付きでログ記録
