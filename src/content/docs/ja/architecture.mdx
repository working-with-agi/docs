## レイヤー構成

### Frontend Layer

あなたのアプリケーションに `@work-with-ai/vue` 等のコンポーネントを配置します。ユーザーはまるでソフトウェアのような対話画面から AI と作業を開始します。

- Vue / React / Angular 対応
- WebSocket で AGI Terminal に直接接続
- xterm.js ベースのレンダリング（実装詳細はユーザーに見えない）

### AGI Terminal (agiterm)

Aether Platform 上の AI エージェント実行環境です。セッションごとにサンドボックスコンテナが起動し、安全に AI エージェントを実行します。

**セッション作成のライフサイクル:**

1. **Provision** — サンドボックスコンテナ / ワークスペースを作成
2. **Inject** — Skill（SKILL.md）とコンテキスト（CLAUDE.md）をワークスペースに注入。認証情報を `.agiterm/credentials/` にファイルとして配置（0600）
3. **Start** — AI CLI をワークスペース内で起動

```
{workspace}/
├── .claude/agents/skills/{provider}/
│   └── SKILL.md
├── .agiterm/credentials/
│   ├── token     (0600)
│   └── api_url   (0600)
└── CLAUDE.md
```

### MAGI Engine

AGI Terminal 内で複数の AI エージェントが異なる視点から協調します。NATS メッセージングによるリアルタイム通信で合議を実現。

- **3つの視点**: 賛成 / 反対 / 中立
- **Pros/Cons 分析**: 多角的に検証された結論を導出
- **NATS 基盤**: エージェント間のリアルタイム通信

### Knowledge Hub

ドキュメントやデータを知識基盤として管理するサブシステムです。

| コンポーネント | 役割 |
|------------|------|
| **Ingestion** | ドキュメント・コード・Wiki の取り込み |
| **Embedding Engine** | テキストをベクトルに変換 |
| **Qdrant** | ベクトルデータベース |
| **Semantic Search** | 意味ベースの検索 |

Knowledge Hub が「何を知っているか」を提供します。

### Skill Registry

Knowledge Hub の上で動くスキルを登録・管理します。スキルは `SKILL.md` として定義され、セッション作成時にワークスペースに注入されます。

Skill Registry が「何ができるか」を定義します。AI は SKILL.md の手順に従ってシステムを操作します。

### Kubernetes Infrastructure

AGI Terminal は Kubernetes 上でスケールアウトします。

- **マルチテナント分離** — テナントごとにリソース制限・ネットワーク分離
- **オートスケーリング** — 需要に応じて AGI Terminal Pod を自動スケール
- **NATS** — 内部通信のみ（インターネット非公開）
