## コンセプト: MAGI — Multi Agent General Instrument

WorkWithAGI は **MAGI（Multi Agent General Instrument）** をコンセプトとしたSDKです。

あなたのサービスにAI対話機能を組み込むことで、ユーザーがAIと対話しながら作業を進められるようになります。「これをやって」と指示するだけで、AIがデータの収集・分析・レポート作成まで一貫して自律的にこなします。

チャットボットとは異なり、AIエージェントが**自律的にタスクを計画・実行**します。作業過程が見え、途中で方向修正もできます。

## 課題

プロダクトに AI 搭載の開発者ツールを追加したい — AI アシスタント付きターミナルや、意図を理解するナレッジ検索。ゼロから構築すると数ヶ月かかり、PTY 処理、ベクトルデータベース、埋め込みモデル、リアルタイム WebSocket プロトコルの専門知識が必要です。

## 解決策

WorkWithAGI SDK は、2つのプロダクション対応コンポーネントをドロップインで提供します:

### AI ターミナル

任意のブラウザで動作する Web ベースターミナル。ユーザーが得るもの:

- フル PTY ターミナル（偽シェルではない）
- コンテキストを理解する AI アシスタント内蔵
- コマンドの脅威検知とフィルタリング
- セッション記録と再生
- カスタマイズ可能なテーマとキーバインド

iframe または JavaScript SDK で埋め込み:

```html
<!-- iframe: 設定不要 -->
<iframe src="https://terminal.workwithagi.dev/embed?key=YOUR_KEY"></iframe>
```

```typescript
// SDK: フルコントロール
import { WorkAGI } from "@working-with-agi/sdk";

const term = WorkAGI.terminal("#container", {
  theme: "dark",
  ai: true,
  recording: true,
  onThreat: (alert) => notify(alert),
});
```

### KnowledgeHub

ドキュメント、コード、Wiki をインデックスし、「何が必要か」を記述して検索:

```typescript
const hub = WorkAGI.knowledgeHub({
  index: "my-project",
});

// ディレクトリを取り込み
await hub.ingest("./src");

// 意図で検索
const results = await hub.search("API タイムアウトのエラーハンドリング");
// → ソース、セクション、関連度スコア、スニペット
```

特徴:
- マルチソース取り込み（コード、ドキュメント、Wiki）
- ファイル監視による自動増分インデックス
- ナレッジベース全体のセマンティック検索
- ソース帰属付きのコンテキスト対応検索

## アーキテクチャ

```
あなたのアプリ
  │
  ├── WorkAGI.terminal()       →  WebSocket (Socket.IO) → PTY + AI エージェント
  │
  └── WorkAGI.knowledgeHub()   →  REST API → 埋め込み → ベクトル DB
```

両コンポーネントともセルフホスト可能。マネージドサービスを選ばない限り、データはあなたのインフラの外に出ません。
