## 課題

プロダクトに AI 搭載の開発者ツールを追加したい — AI アシスタント付きターミナルや、意図を理解するインテリジェントなナレッジ検索。ゼロから構築すると数ヶ月かかり、PTY 処理、ベクトルデータベース、埋め込みモデル、リアルタイム WebSocket プロトコルの専門知識が必要です。

## 解決策

WorkWithAGI SDK は、2つのプロダクション対応コンポーネントをドロップインで提供します:

### AI ターミナル

任意のブラウザで動作する Web ベースターミナル。ユーザーが得るもの:

- フル PTY ターミナル（偽シェルではない）
- コンテキストを理解する AI アシスタント内蔵
- コマンドの脅威検知とフィルタリング
- セッション記録と再生
- カスタマイズ可能なテーマとキーバインド

iframe または JavaScript SDK で埋め込み:

```html
<!-- iframe: 設定不要 -->
<iframe src="https://terminal.workwithagi.dev/embed?key=YOUR_KEY"></iframe>
```

```typescript
// SDK: フルコントロール
import { WorkAGI } from "@work-with-ai/sdk";

const term = WorkAGI.terminal({
  container: "#container",
  endpoint: "https://your-server:8600",
  theme: "dark",
  onConnect: () => console.log("Connected"),
  onDisconnect: () => console.log("Disconnected"),
});
```

### KnowledgeHub

ドキュメント、コード、Wiki をインデックスし、正確な名前を覚えていなくても必要なものを記述して検索:

```typescript
import { KnowledgeClient } from "@work-with-ai/sdk";

const kb = new KnowledgeClient("https://your-hub:8610", "your-api-key");

// 意図で検索
const results = await kb.search("error handling for API timeouts");
// → ソース、セクション、関連度スコア、スニペット
```

特徴:
- マルチソース取り込み（コード、ドキュメント、Wiki）
- ファイル監視による自動増分インデックス
- ナレッジベース全体のセマンティック検索
- ソース帰属付きのコンテキスト対応検索

## アーキテクチャ

```
あなたのアプリ
  │
  ├── WorkAGI.terminal()       →  WebSocket (Socket.IO) → PTY + AI エージェント
  │
  └── KnowledgeClient          →  REST API → 埋め込み → ベクトル DB
```

両コンポーネントともセルフホスト可能。マネージドサービスを選ばない限り、データはあなたのインフラの外に出ません。
